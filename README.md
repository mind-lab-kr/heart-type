# 나의 애착유형 심리테스트

> 나는 사랑을 어떻게 받고 싶은 걸까?

20가지 연애 상황에 대한 선택을 바탕으로 **16가지 애착유형** 중 나의 유형을 찾아주는 감성 심리테스트 웹사이트입니다. 순수 HTML/CSS/JS로 제작되어 별도 빌드 없이 정적 호스팅(GitHub Pages 등)으로 바로 배포할 수 있습니다.

## 파일 구성

```
heart-type/
├── index.html   # 인트로 + 20문항 퀴즈 화면
├── result.html  # 6단계 슬라이드 결과 화면
├── style.css    # 핑크/퍼플 그라데이션 + 글래스모피즘 + 파티클 애니메이션
├── data.js      # 16가지 유형 데이터, 20문항, 쿠팡 링크
├── app.js       # 퀴즈 로직, 채점, 슬라이드 네비게이션, 쿠팡 창 감지, 공유 기능
└── README.md
```

`heart-thumbnail.jpg`는 OG(오픈그래프) 썸네일 이미지 파일명 자리표시자입니다. 프로젝트 루트(`heart-type/`)에 실제 썸네일 이미지를 `heart-thumbnail.jpg`라는 이름으로 추가해주세요. (권장 사이즈: 1200x630)

## 애착유형 & 채점 방식

20문항 × 4개 보기(A/B/C/D)로 구성되며, 보기는 항상 다음 네 가지 성향 중 하나를 대표합니다.

| 보기 | 성향 | 하위 유형 4가지 |
|---|---|---|
| A | 불안 애착 | 집착형 · 눈치형 · 감정폭발형 · 완벽추구형 |
| B | 회피 애착 | 냉담형 · 바쁜척형 · 독립형 · 분석형 |
| C | 안정 애착 | 리더형 · 배려형 · 자유형 · 성장형 |
| D | 혼란 애착 | 폭풍형 · 무감각형 · 자기방어형 · 이중감정형 |

각 문항의 A/B/C/D 보기는 4가지 하위 유형을 20문항에 걸쳐 균등하게 순환하며 태깅되어 있어(하위 유형당 정확히 5회), 20개 응답 중 가장 많이 선택된 하위 유형이 최종 결과로 계산됩니다. 동점일 경우 `TYPE_IDS` 배열 순서상 먼저 나오는 유형이 선택됩니다.

결과는 `localStorage`에 저장되어 `result.html`로 이동한 뒤 표시됩니다.

## 결과 페이지 슬라이드 구성

1. **기본 결과** (무료 공개) — 유형/이모지/한줄설명/핵심 특징 3가지
2. **잠금 해제 대기** — 쿠팡 링크 오픈 후 창이 닫히길 대기 (500ms 간격 감지)
3. **심층 분석** — 근본 원인, 연애 패턴, 5가지 스탯 그래프
4. **궁합 & 주의사항** — 베스트 궁합 TOP2, 주의 유형, 연애 시 주의점 3가지
5. **극복 & 성장 로드맵** — 장단점, 안정형으로 가는 실천법 3가지, 위로 메시지
6. **공유 페이지** — 결과 요약 카드, 카카오톡 공유, 링크 복사, 다시 테스트하기

## 쿠팡 파트너스 연동

- `data.js`의 `COUPANG_LINKS`에 16개 유형별 메인/백업 링크가 등록되어 있습니다.
- 슬라이드1의 "더 자세한 내 마음의 지도 보기" 버튼 클릭 시 메인 링크를 새 창으로 엽니다.
- 팝업이 차단되었거나 메인 링크가 열리자마자 닫히면 자동으로 백업 링크로 전환합니다.
- 새 창이 닫히는 것을 500ms 간격으로 감지해 자동으로 슬라이드3(잠금 해제)으로 이동합니다.
- 쿠팡 파트너스 고지 문구가 모든 페이지 하단(footer)에 표시됩니다.

## 카카오톡 공유 설정 (선택)

`data.js` 상단의 `KAKAO_JS_KEY` 값을 [Kakao Developers](https://developers.kakao.com)에서 발급받은 JavaScript 키로 교체하면 슬라이드6의 "카카오톡으로 공유하기" 버튼이 실제 카카오톡 공유로 동작합니다. 키를 설정하지 않으면 버튼 클릭 시 자동으로 링크가 복사되고 안내 메시지가 표시됩니다.

카카오 개발자 콘솔의 **내 애플리케이션 > 플랫폼 > Web**에 배포한 GitHub Pages 도메인을 등록해야 합니다.

## 로컬에서 실행하기

정적 파일이므로 `file://`로 직접 열어도 되지만, `fetch`/모듈 관련 제약을 피하려면 간단한 로컬 서버 사용을 권장합니다.

```bash
cd heart-type
python -m http.server 8080
# 브라우저에서 http://localhost:8080/index.html 접속
```

## GitHub Pages 배포

모든 리소스 경로는 상대경로(`./style.css`, `./data.js` 등)로 작성되어 있어 리포지토리의 어떤 하위 경로에 배포해도 정상 동작합니다.

1. 이 `heart-type` 폴더를 GitHub 리포지토리에 푸시합니다.
2. 리포지토리 **Settings > Pages**에서 배포 브랜치(예: `main`)와 폴더를 지정합니다.
   - 리포지토리 루트가 곧 `heart-type`이라면 `/ (root)` 선택
   - 다른 프로젝트와 함께 있다면 `heart-type` 폴더를 별도 리포지토리로 분리하거나, Pages 배포 시 해당 경로를 포함하도록 설정
3. `heart-thumbnail.jpg`를 추가하는 것을 잊지 마세요.
4. 배포 후 `https://<username>.github.io/<repo>/index.html` 로 접속해 확인합니다.
